8. Error Handling và Rollback

Xử lý Lỗi và Rollback Strategies

Trong phần này, chúng ta sẽ tìm hiểu về các chiến lược xử lý lỗi và rollback trong microservices deployment orchestration.

Mục tiêu phần này:

  • Hiểu các loại lỗi khác nhau trong microservices deployment
  • Implement comprehensive error handling strategies
  • Thiết kế rollback mechanisms
  • Test error scenarios và validation

8.1 Error Handling Strategies

Types of Errors trong Microservices Deployment

1. Transient Errors (Có thể retry)

  • Network timeouts
  • Temporary service unavailability
  • Resource throttling
  • Database connection issues

2. Permanent Errors (Cần rollback)

  • Configuration errors
  • Code deployment failures
  • Data corruption
  • Permission denied

3. Partial Failures (Cần decision logic)

  • Some services succeed, others fail
  • Degraded performance
  • Dependency unavailability

8.2 Comprehensive Error Handling Implementation

Enhanced Lambda Error Handling

Updated Health Check Function:

export const handler = async (event) => {
    const { serviceName, serviceUrl, simulateFailure } = event;
    
    try {
        // Check for failure simulation
        if (simulateFailure) {
            return await simulateErrorScenario(simulateFailure, serviceName);
        }
        
        // Normal health check logic
        const result = await performHealthCheck(serviceName, serviceUrl);
        return result;
        
    } catch (error) {
        // Enhanced error reporting
        const errorResponse = {
            statusCode: 500,
            body: {
                serviceName,
                status: 'UNHEALTHY',
                error: {
                    type: error.name || 'UnknownError',
                    message: error.message,
                    code: error.code,
                    retryable: isRetryableError(error),
                    timestamp: new Date().toISOString()
                }
            }
        };
        
        console.error(`❌ Health check failed for ${serviceName}:`, errorResponse);
        
        // For certain errors, throw to trigger Step Functions retry
        if (isRetryableError(error)) {
            throw new Error(`Retryable error in ${serviceName}: ${error.message}`);
        }
        
        return errorResponse;
    }
};

async function simulateErrorScenario(simulateFailure, serviceName) {
    const { errorType, errorCode } = simulateFailure;
    
    console.log(`🧪 Simulating ${errorType} error for ${serviceName}`);
    
    switch (errorType) {
        case 'timeout':
            // Simulate timeout
            await new Promise(resolve => setTimeout(resolve, 35000));
            break;
            
        case 'service_unavailable':
            throw new Error(`Service ${serviceName} is temporarily unavailable`);
            
        case 'permission_denied':
            const permError = new Error(`Permission denied for ${serviceName}`);
            permError.code = 'PERMISSION_DENIED';
            permError.retryable = false;
            throw permError;
            
        case 'resource_exhausted':
            const resourceError = new Error(`Resource limit exceeded for ${serviceName}`);
            resourceError.code = 'RESOURCE_EXHAUSTED';
            resourceError.retryable = true;
            throw resourceError;
            
        default:
            throw new Error(`Unknown error type: ${errorType}`);
    }
}

function isRetryableError(error) {
    const retryableCodes = [
        'ECONNRESET',
        'ENOTFOUND', 
        'TIMEOUT',
        'RESOURCE_EXHAUSTED',
        'THROTTLING'
    ];
    
    return retryableCodes.includes(error.code) || error.retryable === true;
}

8.3 Rollback Coordination

Rollback Strategy Design

1. Rollback Principles

  • Reverse dependency order: Rollback theo thứ tự ngược lại
  • Graceful degradation: Giảm tải từ từ
  • Data consistency protection: Bảo vệ tính nhất quán dữ liệu
  • Minimal downtime: Tối thiểu thời gian downtime

2. Rollback Types

  • Hot Rollback: Switch traffic to previous version
  • Cold Rollback: Stop và redeploy previous version
  • Partial Rollback: Rollback specific services only

Enhanced Rollback Coordinator

export const handler = async (event) => {
    const { 
        failedService,
        deployedServices = [],
        rollbackReason,
        rollbackStrategy = 'reverse-dependency'
    } = event;
    
    console.log(`🔄 Initiating rollback coordination`);
    console.log(`❌ Trigger: ${failedService} failed - ${rollbackReason}`);
    
    try {
        // Determine rollback scope
        const rollbackPlan = await createRollbackPlan(failedService, deployedServices);
        
        // Execute rollback phases
        const rollbackResults = await executeRollbackPlan(rollbackPlan);
        
        // Validate rollback success
        const validationResults = await validateRollback(rollbackPlan.services);
        
        return {
            statusCode: 200,
            body: {
                rollbackStatus: 'COMPLETED',
                rollbackPlan,
                rollbackResults,
                validationResults,
                completedAt: new Date().toISOString()
            }
        };
        
    } catch (error) {
        console.error('❌ Rollback coordination failed:', error);
        
        return {
            statusCode: 500,
            body: {
                rollbackStatus: 'FAILED',
                error: error.message,
                partialResults: error.partialResults || [],
                failedAt: new Date().toISOString()
            }
        };
    }
};

async function createRollbackPlan(failedService, deployedServices) {
    console.log(`📋 Creating rollback plan for failure: ${failedService}`);
    
    // Service dependency mapping
    const dependencies = {
        'review': ['order', 'auth'],
        'order': ['menu', 'auth'], 
        'menu': ['auth'],
        'restaurant': ['auth'],
        'auth': []
    };
    
    // Determine which services need rollback
    const servicesToRollback = [];
    
    // If auth fails, rollback everything
    if (failedService === 'auth') {
        servicesToRollback.push(...deployedServices);
    } else {
        // Rollback failed service and its dependents
        servicesToRollback.push(failedService);
        
        // Find dependent services
        for (const [service, deps] of Object.entries(dependencies)) {
            if (deps.includes(failedService) && deployedServices.includes(service)) {
                servicesToRollback.push(service);
            }
        }
    }
    
    // Order by reverse dependency (most dependent first)
    const rollbackOrder = ['review', 'order', 'restaurant', 'menu', 'auth']
        .filter(service => servicesToRollback.includes(service));
    
    const rollbackPlan = {
        failedService,
        services: rollbackOrder,
        strategy: 'reverse-dependency',
        phases: rollbackOrder.map((service, index) => ({
            phase: index + 1,
            service,
            action: 'rollback',
            dependencies: dependencies[service]
        }))
    };
    
    console.log(`📋 Rollback plan:`, JSON.stringify(rollbackPlan, null, 2));
    return rollbackPlan;
}

async function executeRollbackPlan(rollbackPlan) {
    console.log(`⚙️ Executing rollback plan for ${rollbackPlan.services.length} services`);
    
    const results = [];
    
    for (const phase of rollbackPlan.phases) {
        console.log(`🔄 Phase ${phase.phase}: Rolling back ${phase.service}`);
        
        try {
            const rollbackResult = await rollbackService(phase.service);
            
            results.push({
                phase: phase.phase,
                service: phase.service,
                status: 'ROLLED_BACK',
                duration: rollbackResult.duration,
                details: rollbackResult.operations,
                timestamp: new Date().toISOString()
            });
            
            // Wait between phases for stability
            if (phase.phase < rollbackPlan.phases.length) {
                console.log(`⏳ Waiting 2 seconds before next phase...`);
                await new Promise(resolve => setTimeout(resolve, 2000));
            }
            
        } catch (error) {
            console.error(`❌ Failed to rollback ${phase.service}:`, error);
            
            results.push({
                phase: phase.phase,
                service: phase.service,
                status: 'ROLLBACK_FAILED',
                error: error.message,
                timestamp: new Date().toISOString()
            });
            
            // Continue with other services despite failure
        }
    }
    
    return results;
}

async function rollbackService(serviceName) {
    const startTime = Date.now();
    console.log(`🔄 Rolling back service: ${serviceName}`);
    
    // Simulate rollback operations
    const operations = [
        { name: `Stop ${serviceName} traffic`, duration: 500 },
        { name: `Drain ${serviceName} connections`, duration: 1000 },
        { name: `Revert ${serviceName} configuration`, duration: 800 },
        { name: `Restart ${serviceName} with previous version`, duration: 1500 },
        { name: `Validate ${serviceName} rollback`, duration: 600 }
    ];
    
    const completedOperations = [];
    
    for (const operation of operations) {
        console.log(`  ⚙️ ${operation.name}...`);
        await new Promise(resolve => setTimeout(resolve, operation.duration));
        completedOperations.push(operation.name);
    }
    
    const duration = Date.now() - startTime;
    
    return {
        success: true,
        duration,
        operations: completedOperations
    };
}

async function validateRollback(services) {
    console.log(`✅ Validating rollback for ${services.length} services`);
    
    const validationResults = [];
    
    for (const service of services) {
        try {
            // Simulate health check on rolled back service
            await new Promise(resolve => setTimeout(resolve, 200));
            
            validationResults.push({
                service,
                status: 'HEALTHY',
                message: `${service} successfully rolled back and operational`
            });
            
        } catch (error) {
            validationResults.push({
                service,
                status: 'UNHEALTHY',
                message: `${service} rollback validation failed: ${error.message}`
            });
        }
    }
    
    return validationResults;
}

8.5 Advanced Error Handling Patterns

Circuit Breaker Pattern

class CircuitBreaker {
    constructor(threshold = 5, timeout = 60000) {
        this.threshold = threshold;
        this.timeout = timeout;
        this.failureCount = 0;
        this.lastFailureTime = null;
        this.state = 'CLOSED'; // CLOSED, OPEN, HALF_OPEN
    }
    
    async execute(operation) {
        if (this.state === 'OPEN') {
            if (Date.now() - this.lastFailureTime > this.timeout) {
                this.state = 'HALF_OPEN';
            } else {
                throw new Error('Circuit breaker is OPEN');
            }
        }
        
        try {
            const result = await operation();
            this.onSuccess();
            return result;
        } catch (error) {
            this.onFailure();
            throw error;
        }
    }
    
    onSuccess() {
        this.failureCount = 0;
        this.state = 'CLOSED';
    }
    
    onFailure() {
        this.failureCount++;
        this.lastFailureTime = Date.now();
        
        if (this.failureCount >= this.threshold) {
            this.state = 'OPEN';
        }
    }
}

Bulkhead Pattern

// Isolate different types of operations
const deploymentPool = new ResourcePool(5); // 5 concurrent deployments
const healthCheckPool = new ResourcePool(10); // 10 concurrent health checks
const rollbackPool = new ResourcePool(3); // 3 concurrent rollbacks

async function deployWithBulkhead(serviceName) {
    return deploymentPool.execute(async () => {
        return await deployService(serviceName);
    });
}

Timeout và Deadline Management

async function executeWithTimeout(operation, timeout = 30000) {
    return Promise.race([
        operation(),
        new Promise((_, reject) => 
            setTimeout(() => reject(new Error('Operation timed out')), timeout)
        )
    ]);
}
