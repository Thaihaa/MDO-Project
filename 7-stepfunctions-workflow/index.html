7. Step Functions Workflow

Thiết lập Step Functions Workflow

Trong phần này, chúng ta sẽ tạo Step Functions workflow để orchestrate việc deployment các microservices theo đúng thứ tự dependency và xử lý các tình huống lỗi.

Mục tiêu phần này:

  • Thiết kế workflow architecture cho microservices deployment
  • Implement parallel và sequential execution
  • Tích hợp error handling và retry mechanisms
  • Tạo comprehensive monitoring và logging

7.1 Workflow Design Principles

Core Principles

  1. Modularity: Mỗi state có single responsibility
  2. Resilience: Built-in error handling và retry logic
  3. Observability: Comprehensive logging và metrics
  4. Scalability: Parallel execution khi có thể

Workflow Architecture

graph TD

    B --> C[Deploy Layer 1: Auth Service]
    C --> D[Health Check Auth]

    F --> G[Deploy Layer 3: Order Service]

    H --> I[Deploy Layer 4: Review Service]
    I --> J[Health Check Review]
    J --> K[Send Success Notification]
    
    D --> L[Rollback Coordinator]
    F --> L
    H --> L
    J --> L
    L --> M[Send Failure Notification]

7.2 State Machine Implementation

Bước 1: Tạo State Machine

  1. Truy cập AWS Console → Services → Step Functions
  2. Click “Create state machine”
  3. Choose “Write your workflow in code”

7.3 Advanced Workflow Features

Enhanced Error Handling

Quan trọng: Implement comprehensive error handling để đảm bảo system reliability.

Advanced Retry Configuration:

{
  "Retry": [
    {
      "ErrorEquals": ["States.TaskFailed"],
      "IntervalSeconds": 2,
      "MaxAttempts": 3,
      "BackoffRate": 2.0,
      "Comment": "Exponential backoff for transient failures"
    },
    {
      "ErrorEquals": ["States.Timeout"], 
      "IntervalSeconds": 1,
      "MaxAttempts": 2,
      "BackoffRate": 1.5,
      "Comment": "Quick retry for timeouts"
    },
    {
      "ErrorEquals": ["Lambda.ServiceException"],
      "IntervalSeconds": 5,
      "MaxAttempts": 2,
      "Comment": "AWS service issues"
    }
  ]
}

Comprehensive Catch Blocks:

{
  "Catch": [
    {
      "ErrorEquals": ["States.TaskFailed"],
      "Next": "HandleServiceFailure",
      "ResultPath": "$.error",
      "Comment": "Catch task failures and route to error handler"
    },
    {
      "ErrorEquals": ["States.Timeout"],
      "Next": "HandleTimeout", 
      "ResultPath": "$.timeoutError"
    },
    {
      "ErrorEquals": ["States.ALL"],
      "Next": "HandleUnknownError",
      "ResultPath": "$.unknownError",
      "Comment": "Fallback for unexpected errors"
    }
  ]
}

7.4 Workflow Configuration và Deployment

Bước 1: Configure State Machine

  1. Name: restaurant-deployment-orchestrator
  2. Type: Standard workflow
  3. Role: Tạo IAM role với permissions:
    • Lambda invoke permissions
    • CloudWatch logs permissions
    • Step Functions execution permissions

Bước 2: IAM Role Configuration

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "lambda:InvokeFunction"

      "Resource": [
        "arn:aws:lambda:*:*:function:restaurant-health-check",

7.5 Monitoring và Observability

CloudWatch Integration

Best Practice: Enable detailed logging để track workflow execution.

Log Configuration:

{
  "level": "ALL",
  "includeExecutionData": true,
  "destinations": [
    {
      "cloudWatchLogsLogGroup": {
        "logGroupArn": "arn:aws:logs:region:account:log-group:/aws/stepfunctions/restaurant-deployment"
      }
    }
  ]
}

Metrics và Alarms

Key Metrics to Monitor:

  • Execution duration
  • Success/failure rates
  • Individual service deployment times
  • Error rates by service
  • Rollback frequency

CloudWatch Alarms:

# Deployment failure rate alarm
aws cloudwatch put-metric-alarm \
  --alarm-name "RestaurantDeploymentFailureRate" \
  --alarm-description "Alert when deployment failure rate exceeds 20%" \
  --metric-name "ExecutionsFailed" \
  --namespace "AWS/States" \
  --statistic "Sum" \
  --period 300 \
  --threshold 2 \
  --comparison-operator "GreaterThanThreshold"
